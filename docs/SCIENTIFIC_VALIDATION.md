# 科研验证协议

## 目标

本协议用于区分“程序内部一致”与“真实田间有效”。只有完成下列外部验证后，结果才能用于论文定量结论或受控田间试验。

## 数据集分层

1. **金标准小样本**：裁剪后的真实 GeoTIFF、固定 CRS、人工核验行带和预期坐标，用于每次版本回归。
2. **跨工况样本**：不同田块、生育期、飞行高度、光照和含水状态。
3. **失败样本**：无 CRS、错误仿射、NoData、四波段、行带断裂、严重粘连和田头不足。
4. **外场执行样本**：规划路径、终端接收文件、RTK 实际轨迹和作业后碾压调查。

原始数据不进入 Git；应单独保存只读数据，并维护文件 SHA-256、采集日期、设备、飞行参数和授权信息。

## 坐标准确性

- 无 GCP/RTK 时只报告 GeoTIFF 自带配准下的内部一致性，不报告绝对定位精度。
- 有控制点后，分别报告平面 RMSE、最大误差、95% 分位误差和异常点处理规则。
- 在目标导航终端中核对经纬度顺序、坐标系、状态码、点序和文件校验值。

## 模型验证

- 固定训练/验证/测试划分及其版本哈希。
- 记录类别映射、输入波段、位深归一化、图像尺寸、置信度和 IoU 阈值。
- 除总体指标外，报告不同工况分组指标和失败案例。

## 发布门禁

- 全量自动测试通过。
- 金标准数据输出在预设容差内。
- `pip check` 无依赖冲突。
- 内置模型哈希与 `data/models/model_registry.json` 一致。
- 导出文件与 manifest 哈希一致。
- 版本标签、Git 提交和实验记录一致。
